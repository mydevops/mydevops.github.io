[Python 2] Python 多线程

Posted on 2017-03-01   |   In python

在CPython解释器中：由于GIL的缘故，同一个进程内同一时刻只允许运行一个线程。如果要利用多核CPU那么建议使用multiprocessing模块。但是IO密集型任务多线程仍然是一个很好的选择。

Python中使用threading模块实现了多线程，threading模块提供了如下的函数和对象。

threading

threading.Thread()

用于在主线程中个创建一个线程，创建的方式有两种：

第一种：重写此类的init()和run()方法。

# -*- coding: utf-8 -*-

import threading

class MyThreading(threading.Thread):

    def __init__(self):
        threading.Thread.__init__(self) # 必须要调用父类的构造函数

    def run(self):
        pass
        # do something

if __name__ == "__main__":
    t = MyThreading()
    t.start()

第二种：直接启动

# -*- coding: utf-8 -*-

import threading

def run():
    pass
    # do something

if __name__ == "__main__""
    t = threading.Thread(target=run)
    t.start()

实例化Thread对象后必须通过start()方法来启动这个线程，线程启动后会自动调用run()方法。

构造函数

init(target=None, name=None, args=(), kwargs={})

  • target：函数名称，在线程启动后执行的函数。
  • name：线程名称，默认是Thread-N格式的名字，N是一个十进制数字。
  • args：target指定函数的位置参数列表，接收一个元组。
  • kwargs：target指定函数的关键字参数列表，接收一个字典。

start()

启动线程。

run()

通过继承方式创建线程时必须重写此方法。一旦启动一个线程之后，就认为这个线程是激活状态，直到run()方法执行完毕或者引发一个未处理异常时此线程被终止。is_alive()方法可以查看这个线程是否处于激活状态。

join([timeout])

等待线程结束，一直阻塞到线程正常结束或者有一个未处理的异常或者超时时间到了。若timeout生效，必须使除主线程以外的其他线程为守护线程。

# -*- coding: utf-8 -*-

import time
import threading

def run(number):
    print "run method start .."
    print "Task ID: {}".format(number)
    time.sleep(2)
    print "run method end .."

if __name__ == "__main__":
    start_time = time.time()
    thread_list = []
    for number in range(1, 51):
        t = threading.Thread(target=run, args=(number,))
        t.start()
        thread_list.append(t)
    [ t.join() for t in thread_list ]
    print "run time: {}".format(time.time() - start_time)

name

线程名称。

getName()

获取线程名称。

setName()

设置线程名称。

ident

线程ID。

isAlive()

返回线程是否激活状态。

daemon

标识线程是否为守护线程：是(True)，否(False)，该方法必须在start()方法前执行。它的初始值是从创建它的线程继承的。主线程默认是daemon = False，所以由主线程创建的线程默认daemon = False。

isDaemon()

判断线程是否是守护线程。

setDaemon()

设置线程是否为守护线程。

# -*- coding: utf-8 -*-

import time
import threading

def run(number):
    print "run method start .."
    print "Task ID: {}".format(number)
    time.sleep(2)
    print "run method end .."

if __name__ == "__main__":
    start_time = time.time()
    thread_list = []
    for number in range(1, 51):
        t = threading.Thread(target=run, args=(number,))
        t.setDaemon(True)
        t.start()
        thread_list.append(t)
    print "run time: {}".format(time.time() - start_time)

threading.enumerate()

返回当前激活状态的Thread对象的列表，包含守护线程和主线程。

threading.activeCount()

返回当前激活状态的Thread对象的数量，这个数量等同于enumerate()方法返回的列表的长度。

threading.Lock()

互斥锁，避免多个线程同时修改同一块数据。

acquire()

加锁。

release()

解锁。

# -*- coding: utf-8 -*-

import threading

def add():
    lock.acquire()
    global number
    number += 1
    lock.release()

if __name__ == "__main__":
    lock = threading.Lock()
    number = 0
    threading_list = []
    for _ in range(10000):
        t = threading.Thread(target=add)
        threading_list.append(t)
        t.start()
    [ t.join() for t in threading_list ]
    print number

threading.RLock()

递归锁。

acquire()

加锁。

release()

解锁。

# -*- coding: utf-8 -*-

import threading

def run2():
    lock.acquire()
    print "run run2 method~"
    lock.release()

def run1():
    lock.acquire()
    print "run run1 method~"
    run2()
    lock.release()

if __name__ == "__main__":
    lock = threading.RLock()
    t = threading.Thread(target=run1)
    t.start()
    t.join()

threading.BoundedSemaphore([value])

信号量，用于控制线程数量。

acquire()

加锁。

release()

解锁。

# -*- coding: utf-8 -*-

import time
import threading

def run(number):
    semaphore.acquire()
    print "Task ID:{}".format(number)
    time.sleep(1)
    semaphore.release()

if __name__ == "__main__":
    semaphore = threading.BoundedSemaphore(5)
    threading_list = []
    for number in range(1,21):
        t = threading.Thread(target=run, args=(number,))
        t.start()
        threading_list.append(t)
    [ t.join() for t in threading_list ]

with

Lock、RLock和BoundedSemaphore对象可以用作with语句的上下文管理器。进入该代码块时将调用acquire() 方法，退出代码块时则调用release()。

threading.Event()

事件对象是线程间最简单的通信机制之一。每个事件对象管理一个内部标志，可以在事件对象上调用set()方法将内部标志设为true，调用clear()方法将内部标志重置为false。在事件对象调用wait()方法时，线程将阻塞直到事件对象的内部标志被设为true。

set()

设置标记为 True。

clear()

设置标记为 False。

wait()

在事件对象调用wait()方法时，线程将阻塞直到内部标志被设置为True。

isSet()

判断标记是否被设置。

# -*- coding: utf-8 -*-

import time
import threading

def light():
    """
    3s 红灯
    2s 绿灯
    1s 黄灯
    """
    global event
    i = 0
    while True:
        if i < 3:
            print "\033[41;1m红灯\033[0m"
            event.clear()
        elif i < 5:
            print "\033[42;1m绿灯\033[0m"
            event.set()
        elif i < 6:
            print "\033[43;1m黄灯\033[0m"
            event.set()
            i = -1
        i += 1
        time.sleep(1)

def car(number):
    global event
    while True:
        if event.isSet():
            print "汽车[{}]奔跑。。".format(number)
        else:
            print "汽车[{}]等待。。".format(number)
        time.sleep(1)

if __name__ == "__main__":
    event = threading.Event()
    l = threading.Thread(target=light)
    l.start()
    for i in range(1,6):
        c = threading.Thread(target=car, args=(i,))
        c.start()

threading.local()

threading.Timer()

定时器，一个线程，它在一个指定的时间间隔之后执行一个函数。

构造函数

init(interval, function, args=[], kwargs={}

  • interval：秒之后执行函数
  • function：执行的函数名称
  • args：位置参数
  • kwargs：关键字参数

cancel()

取消定时器。

# -*- coding: utf-8 -*-

import threading

def run():
    print "hello world~"

if __name__ == "__main__":
    timer = threading.Timer(2, run)
    timer.start()

线程间通信

queue模块实现了多生产者、多消费者队列。共实现了三类队列，主要差别在于取得数据的顺序上。

  1. Queue(maxsize=0)：在FIFO(First In First Out，先进先出)队列中，最早加入的任务会被最先得到。
  2. LifoQueue(maxsize=0)：在LIFO(Last In First Out，后进先出)队列中，最后加入的任务会被最先得到(就像栈一样)。
  3. PriorityQueue(maxsize=0)：在优先队列中，任务被保持有序，拥有最小值的任务（优先级最高）被最先得到。

maxsize是个整数，指明了队列中能存放的数据个数的上限。一旦达到上限，插入会导致阻塞，直到队列中的数据被消费掉。如果maxsize小于或者等于0，队列大小没有限制。

Queue对象常用方法(Queue、LifoQueue、PriorityQueue公用方法)：

qsize()

队列的大小。

empty()

如果队列为空返回True，否则返回False。

full()

如果队列是满的返回True，否则返回False。

put(item[, block[, timeout]])

将item放入队列中。

如果block为True且timeout为空对象那么阻塞调用，无超时。阻塞调用线程，直到有空闲槽可用。

如果block为False，如果有空闲槽可用将数据放入队列，否则立即抛出Full异常(非阻塞调用，timeout被忽略)。

如果timeout是个正整数，阻塞调用进程最多timeout秒，如果一直无空闲槽可用，抛出Full异常(带超时的阻塞调用)。

put_nowait(item)

等同于put(item, False)(非阻塞调用)。

get([block[, timeout]])

从队列中移除并返回一个数据。

如果block为True且timeout为空对象阻塞调用，无超时，阻塞调用进程直到有数据可用。

如果block为False，如果有数据可用返回数据，否则立即抛出Empty异常(非阻塞调用，timeout被忽略)。 如果timeout是个正整数，阻塞调用进程最多timeout秒，如果一直无数据可用，抛出Empty异常（带超时的阻塞调用）。

get_nowait()

等同于get(False)(非阻塞调用)。

task_done()

意味着之前入队的一个任务已经完成。由队列的消费者线程调用。每一个get()调用得到一个任务, 接下来的 task_done()调用告诉队列该任务已经处理完毕。

如果当前一个join()正在阻塞，它将在队列中的所有任务都处理完时恢复执行(即每一个由put()调用入队的任务都有一个对应的task_done()调用)。

join()

阻塞调用线程，直到队列中的所有任务被处理掉。

只要有数据被加入队列，未完成的任务数就会增加。当消费者线程调用task_done()(意味着有消费者取得任务并完成任务)，未完成的任务数就会减少。当未完成的任务数降到0，join()解除阻塞。

线程池

虽然线程的开销相对于相对进程而言是微小的，但是如果频繁的创建和销毁线程的这部分损耗可以被避免掉，对于性能的提升是很有必要的。

实现线程池

线程池简单来说就是线程+队列。

# -*- coding: UTF-8 -*-

import time
import Queue
import threading

class Worker(threading.Thread):

    def __init__(self, queue):
        super(Worker, self).__init__()
        self.__queue = queue
        self.setDaemon(True)
        self.start()

    def run(self):
        while True:
            function, args, kwargs = self.__queue.get()
            function(*args, **kwargs)
            print "Thread Name: {}".format(self.name)
            self.__queue.task_done()

class ThreadPool(object):

    def __init__(self, num_workers):
        self.__queue = Queue.Queue(num_workers)
        [ Worker(self.__queue) for _ in range(num_workers) ]

    def spawn(self, f, *args, **kwargs):
        self.__queue.put((f, args, kwargs))

    def wait(self):
        self.__queue.join()

def test_function(number):
    time.sleep(1)
    return number

if __name__ == "__main__":
    pool = ThreadPool(5)985 211
    for number in range(12):
        pool.spawn(test_function, number)

    pool.wait()

Python 的线程池

Python中有个需要安装的threadpool线程池模块，但是已经不推荐使用，官方建议使用标准库中的multiprocessing模块或者asyncio模块来替代。asyncio是Python3新增的模块，这里暂不做讨论。
